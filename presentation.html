Why We Used the Waterfall Model

The Waterfall model is used for the Salon Commission and Sales Management System because it follows a step-by-step process. In this system, the requirements are already clear, like tracking sales, computing commissions, and generating reports. Since we already know what the system should do, it’s easier to follow the Waterfall model where we finish one phase before moving to the next. Benefits of the Waterfall Model

  • Ensure thorough documentation and planning before development began
  • Minimize changes and rework by clarifying requirements up front
  • Track progress easily through distinct, well-defined phases

By using the Waterfall model, we were able to maintain a disciplined workflow and meet our project objectives efficiently.

SDLC Phase 4: Testing

In this phase, we checked if the system works correctly and meets the requirements. We performed several types of testing:

  • ✅ Unit Testing: Tested individual modules (Employee, Sales, Commission, Reports).
  • ✅ Integration Testing: Tested combined modules to ensure smooth system workflow.
